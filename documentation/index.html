Corevex Documentation

Get started with Corevex

Everything you need to install, configure, customize and extend your Laravel 12 + Tailwind admin panel — from first clone to production deploy.

Introduction

Corevex is a premium admin template built on Laravel 12, Tailwind CSS v4 and Alpine.js. It ships role-based access control (Spatie), 25+ reusable Blade components (including a feature-rich Tailwind data table) and fully-designed modules (Dashboard, Inventory, POS, Accounting, Reports, Settings, Users). Interactivity that older admin themes handled with jQuery is now driven by lightweight Alpine directives, and styling is 100% utility-first Tailwind.

Tech stack: Laravel 12 · PHP 8.2+ · Tailwind CSS v4 · Alpine.js · Vite · Spatie Laravel Permission · Laravel Passport · MySQL 8.

What's included

  • Full Laravel 12 application source code
  • 25+ Tailwind + Alpine Blade components in resources/views/components
  • Pre-built modules & screens, all responsive and dark-mode ready
  • Roles, permissions and users management with a redesigned permission picker
  • Live theme customizer with 8 preset palettes
  • Docker configuration for a one-command local environment
  • This documentation and a marketing landing page (in /docs)

Requirements

To run Corevex locally you'll need either Docker (recommended, batteries included) or a local PHP toolchain:

  • PHP 8.2 or higher (with pdo_mysql, mbstring, openssl, gd)
  • Composer 2.x
  • Node.js 18+ and npm (for Vite asset building)
  • MySQL 8 (or MariaDB 10.6+)
  • Docker & Docker Compose — optional but recommended

Quick start (Docker)

The fastest path. From the project root:

# 1 · Create your environment file
cp .env.example .env

# 2 · Start the stack (PHP-FPM, Nginx, MySQL)
docker compose up -d

# 3 · Install PHP dependencies
docker compose exec php composer install

# 4 · Generate the app key
docker compose exec php php artisan key:generate

# 5 · Run migrations + seeders (creates demo data & the admin user)
docker compose exec php php artisan migrate --seed

# 6 · Build front-end assets
npm install && npm run build

The app is now served at http://localhost:8900.

Note: the php container loads .env via env_file, so values become real OS environment variables at container creation. If you change APP_NAME (or other env vars) later, recreate the container so it picks them up: docker compose up -d --force-recreate php.

Manual install (no Docker)

# Install dependencies
composer install
npm install

# Environment
cp .env.example .env
php artisan key:generate

# Configure your database in .env, then:
php artisan migrate --seed

# Build assets (or use `npm run dev` while developing)
npm run build

# Serve
php artisan serve   # http://127.0.0.1:8000

Configuration

Core settings live in .env. The brand name, tagline and meta description are surfaced across titles and auth screens:

APP_NAME=Corevex
APP_TAGLINE="All-in-One Laravel Admin Template for POS, Inventory & Accounting"
APP_VERSION=4.0.0

DB_CONNECTION=mysql
DB_HOST=mysql          # use 127.0.0.1 for a local (non-Docker) DB
DB_PORT=3306
DB_DATABASE=radmin
DB_USERNAME=root
DB_PASSWORD=secret

config/app.php exposes name, tagline and description, all overridable via the matching env keys. Use them anywhere with config('app.tagline').

Building assets

Front-end assets are compiled with Vite. Tailwind v4 is configured via @theme tokens inside resources/css/app.css — there is no tailwind.config.js.

npm run dev      # hot-reloading dev server while you work
npm run build    # optimized production build → public/build

Layouts

Authenticated pages extend the main shell, which wires the sidebar, header, footer and global overlays:

@extends('layouts.main')
@section('title', 'Users')

@section('content')
    {{-- your page content --}}
@endsection

Guest screens (login, register, 404…) use the <x-auth-layout> component, which centers a card and pulls the brand logo automatically.

Navigation & menus

All navigation is data-driven from config/menu.php. The top array powers the topbar workspace switcher; main (and per-module keys) power the sidebars.

'main' => [
    ['label' => 'Dashboard', 'icon' => 'ik ik-home', 'route' => 'dashboard', 'active' => 'dashboard'],

    ['heading' => 'Administrator'],
    ['label' => 'Roles', 'icon' => 'ik ik-award', 'url' => 'roles',
     'active' => 'roles*', 'can' => 'manage_roles'],

    // collapsible group
    ['label' => 'Reports', 'icon' => 'ik ik-trending-up', 'active' => 'reports*', 'children' => [
        ['label' => 'Sales', 'url' => 'reports/sales', 'active' => 'reports/sales'],
    ]],
],

Supported keys: label, icon, route/url, active (a request()->is() pattern), can (gate), badge, desc and children.

Theming & branding

Colors

Brand colors are CSS variables defined under @theme in resources/css/app.css. Edit the --color-primary-* scale to rebrand globally:

@theme {
    --color-primary-500: #007bff;   /* brand */
    --color-primary-600: #006fe6;   /* hover */
    --color-accent-500:  #eb525d;   /* active accent */
    /* sidebar / topbar tokens, gray scale, dark-mode overrides… */
}

Live theme customizer

Open it from the droplet icon in the header. Users can pick from 8 preset palettes or tune the sidebar, topbar and primary color individually. Choices are stored in localStorage and applied before paint (no flash) — no rebuild required.

Logo

The brand mark is a single component, <x-brand-logo>, and the SVG file public/img/corevex-mark.svg (also the favicon). Swap them to use your own brand:

<x-brand-logo markClass="h-8 w-8" textClass="text-lg" />
<x-brand-logo :wordmark="false" />   {{-- mark only --}}

Permission selector

A drop-in picker that auto-groups permissions by their {action}_{resource} name, with search, per-group select-all and a live counter — built to scale past 100 permissions.

<x-permission-selector
    :permissions="$permissions"   {{-- id => name (e.g. Permission::pluck('name','id')) --}}
    :selected="$rolePermissionIds" {{-- edit mode --}}
    name="permissions"             {{-- submits as permissions[] --}}
    channel="role-perms" />        {{-- set selection externally via a window event --}}

Overlays & feedback

Drawers, modals, toasts and the confirm dialog are event-driven through Alpine — dispatch a window event to open them from anywhere.

{{-- Slide-over drawer --}}
<x-drawer name="user-form" title="New user" width="w-[42rem]">
    <form>...</form>
    <x-slot:footer><x-button type="submit">Save</x-button></x-slot:footer>
</x-drawer>
{{-- open it: --}}
<button @click="$dispatch('open-drawer', 'user-form')">New</button>

{{-- Toast (global store) --}}
<button @click="$store.toast.push('Saved!', 'success')">Save</button>

{{-- Confirm dialog --}}
@click="$store.confirm.open({ title: 'Delete?', onConfirm: () => $root.submit() })"

Also included: <x-modal>, <x-dropdown>, <x-tabs>, <x-alert>, <x-empty-state>, <x-skeleton>, <x-avatar>, <x-tooltip>, <x-save-bar> and <x-command-palette> (⌘K).

Roles & permissions

Access control uses Spatie Laravel Permission. Permissions follow a {action}_{resource} convention (e.g. manage_user) so the permission selector can group them automatically.

Protect a route or UI element with the standard gate helpers:

// routes / controllers
$this->middleware('can:manage_user');

{{-- blade --}}
@can('manage_roles')
    <a href="{{ url('roles') }}">Roles</a>
@endcan

Add a new permission from the Permissions screen (a drawer with a colored role picker), or via a seeder/tinker:

use Spatie\Permission\Models\Permission;
Permission::create(['name' => 'manage_invoice']);

Assign permissions to a role from the Roles screen — the create/edit drawer renders <x-permission-selector> and persists with $role->syncPermissions(...).

Add a new page

  1. Route — register it in routes/web.php:
Route::get('/projects', [ProjectController::class, 'index'])->name('projects');
  1. View — create resources/views/projects.blade.php extending the shell:
@extends('layouts.main')
@section('title', 'Projects')
@section('content')
    <x-page-header title="Projects" icon="ik ik-folder" />
    <x-table :paginator="$projects" search> ... </x-table>
@endsection
  1. Menu — add an entry to config/menu.php so it appears in the sidebar.

REST API

Corevex exposes token-authenticated endpoints via Laravel Passport for users, roles and permissions (see app/Http/Controllers/Api and routes/api.php). Authenticate, then call the resources with a Bearer token:

curl -X POST https://your-app.test/api/login \
  -H "Accept: application/json" \
  -d "email=admin@test.com&password=1234"

curl https://your-app.test/api/users \
  -H "Authorization: Bearer <token>"

Deployment

# On the server
composer install --no-dev --optimize-autoloader
npm install && npm run build
php artisan migrate --force

# Cache for production
php artisan config:cache
php artisan route:cache
php artisan view:cache
  • Set APP_ENV=production and APP_DEBUG=false.
  • Point your web server's document root at /public.
  • Ensure storage and bootstrap/cache are writable.

Troubleshooting

Changing APP_NAME in .env has no effect

In Docker, env vars are injected at container creation. Recreate it: docker compose up -d --force-recreate php. Also clear config cache: php artisan config:clear.

Vite manifest not found / unstyled pages

Run npm run build (production) or npm run dev (development).

Blade/CSS changes not showing

Clear caches: php artisan view:clear and rebuild assets if you edited classes.
